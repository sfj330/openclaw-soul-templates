# Project Explanation

## Overview

`openclaw-soul-templates` is a repository of reusable `SOUL.md` templates for `OpenClaw`.

The goal is not to create flashy personalities. The goal is to provide reliable steering layers for different workflows so the agent behaves in a more stable, more useful, and more context-appropriate way over long-term use.

## Why multiple SOUL templates

One generic `SOUL.md` is usually not enough.

Different workflows need different defaults:

- debugging needs root-cause thinking and validation
- writing needs tone control and revision discipline
- research needs evidence awareness and structured comparison
- operations needs risk control, rollback thinking, and prioritization
- customer-facing work needs clarity, empathy, and closure

This repository separates those working modes into individual templates so users can swap them quickly or build their own versions on top.

## Shared structure

Most templates follow the same backbone:

1. Role definition
2. Core principles
3. Default workflow
4. Boundaries and risk control
5. Output style
6. Long-term evolution

This makes the repository easier to understand, compare, and extend.

## Why English-first

The templates in this project are written in English on purpose.

For OpenClaw-style steering files, English instructions tend to be more consistent across models, easier to share publicly, and easier to reuse in mixed-language environments. Users can still localize or personalize the final `SOUL.md` later if needed.

## Who this is for

- OpenClaw users who want a better long-term assistant setup
- people who switch between multiple work modes
- builders who want a GitHub-ready SOUL template library
- teams that want a base set of reusable agent identities

## Value of the repository

This project already includes:

- a clean naming scheme
- a clear folder layout
- 22 practical templates
- a simple PowerShell installer
- documentation that explains how and why to use the pack

## Future extensions

- bilingual editions
- ultra-short and extended variants of each template
- template sets for more industries
- companion packs for `AGENTS.md` and `TOOLS.md`
- backup-aware install and switch scripts
